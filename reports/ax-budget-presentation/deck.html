① 앱

어떤 생각으로 설계했나

MVVM + Repository 화면 / 로직 / 데이터를 분리한 구조

  • 단방향 데이터 흐름 — DB → ViewModel → 화면, 한 방향으로만. 버그가 숨을 곳이 없습니다
  • 화면은 DB를 직접 안 봅니다 — 도메인 모델만
  • 색 · 글꼴은 전부 디자인 토큰(변수) — 어긋날 수 없는 구조

제일 공들인 기능 정기 거래

  • 한 번 등록하면 매달 자동 등록, 앱을 켤 때마다 알아서 연장
  • 수정 · 삭제는 "이번 달만 / 이후 / 전체" 중 선택

명세에 없던 것들 추측 대신 결정

  • 수입엔 카테고리가 없습니다 — 기본 카테고리 7개가 전부 지출용이라서
  • 비율(%)은 반올림 그대로 — 디자이너 코멘트의 정책을 따름

명세의 빈칸은 추측으로 채우지 않고 — 근거를 적은 결정으로 채웠습니다.

① 앱

디테일까지 챙겼습니다

만지는 재미

  • 통계 도넛을 누르고 문지르면 카테고리가 실시간으로 바뀝니다 (진동 피드백)
  • 화면을 옆으로 쓸면 이전 · 다음 주기로 이동

자연스러운 움직임

  • 금액 숫자가 차오르고, 리스트가 차례로 등장
  • 화면 전환 · 저장 · 삭제마다 부드러운 애니메이션과 알림

꼼꼼함

  • 가로로 돌려도 안 깨집니다
  • 처음 쓰는 사람을 위한 가이드 투어 내장

오늘 아침, 발표일에 맞춰 주기가 자동으로 넘어가고 월급 · 월세가 미리 등록돼 있었습니다 — 만든 기능이 실전에서 작동했습니다.

② 어떻게 만들었나

이 앱의 코드는
전부 AI가 작성했습니다

개발 도구는 Claude Code — 저는 코드를 친 게 아니라 AI가 일할 수 있는 환경을 설계하고,
개발하다 막히는 곳마다 직접 도구를 만들어 뚫었습니다. 지금부터 그 과정입니다.

② 어떻게 만들었나

AI에게 무엇을 쥐여줬나

MCP 서버 AI에게 손과 눈을 달아주는 도구 연결 표준

  • figma — 디자인을 그림이 아니라 수치 데이터로 읽기
  • mobile-mcp — 갤럭시 실기기 직접 조작 (터치 · 화면 확인)
  • context7 — 라이브러리 최신 공식 문서 검색

스킬 · 에이전트 일하는 순서의 표준화

  • "화면 만들어줘" 한마디 → 항상 같은 3단계 파이프라인이 돌도록 절차를 스킬로 고정
  • 단계마다 그 일만 하는 전담 서브에이전트

설정 · 훅 자동으로 실행되는 안전장치

  • 위험한 명령(삭제 등)은 무조건 차단
  • 빌드 · 테스트 같은 일상 작업은 승인 없이 자동
  • 설정 전체를 git으로 백업 — 언제든 롤백

핵심 발상 — AI에게 일을 시키는 게 아니라, AI가 일할 수 있는 공장을 짓는다.

사람이 하는 일은 셋 — 링크 주기, 질문에 답하기, 최종 확인. 결정은 사람, 노동은 시스템.

② 어떻게 만들었나

이 시스템은 스스로 좋아집니다

언제무슨 일이 일어나나
도구가 실패하면훅이 실패를 백로그에 자동 기록하고, 다음 세션의 AI가 원인을 제거 — 실패가 곧 개선 과제
작업을 시작할 때마다시스템 이상을 자동 점검 · 경보
매일 밤AI가 혼자 신기술 · 새 도구를 조사해두면, 아침에 사람이 검토해서 반영
3일마다설정 · 문서 전체를 자가 감사

발표 전날 "스스로 검증해봐" 한마디에 — AI가 에이전트 3개로 자기 자신을 교차 검증해서 숨은 버그 5개를 찾아 고쳤습니다.
