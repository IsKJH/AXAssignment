이 실험을 하는 이유

같은 스펙을 주면 AI마다 다른 결정을 내립니다. 그 차이는 코드를 처음 만들 때는 잘 보이지 않습니다. 그런데 그 코드 위에 기능을 추가하고, 실제 API를 붙이고, 단위 테스트를 작성하고, 버그를 고치는 작업이 이어지면 초기 결정의 파급효과가 점점 드러납니다. 실제 API 연동 단계(V4)에서 Claude만 네트워크 로직을 별도 클래스로 분리했고, 그 한 줄의 설계 차이가 이후 테스트 단계(V5)에서 코드량을 Claude 140줄 · Codex 245줄로 갈라놓았습니다. 백그라운드 알림 단계(V8)에서 Gemini는 빌드를 통과하고 실기기에서 즉시 크래시했습니다. 이 실험은 이런 차이가 어디서 생기고 어떻게 누적되는지 추적하기 위해, 동일한 날씨 앱에 동일한 프롬프트를 주고 앱 생성부터 상호 코드 리뷰까지 9단계를 연속으로 진행했습니다.

종합 요약 — V1 → V9

속도 · 구현력

Gemini가 V1~V6 최단(평균 217초). V3에서 날씨 데이터를 자연어로 해석해주는 인사이트 카드와 온도 기반 배경색 같은 스마트 기능을 만들었고, V4 실 API 연동 후에도 이 기능들을 유일하게 유지했습니다. V7부터는 Codex에게 역전됐습니다.

아키텍처 · 설계

Claude의 API 연동 단계(V4) Repository 분리가 단위 테스트 단계(V5)를 140줄로 압축했습니다. Repository는 API 호출 로직을 ViewModel(화면 로직을 담당하는 클래스)과 분리해 테스트에서 가짜 데이터(목 데이터·테스트용 가짜 데이터)로 교체할 수 있게 해주는 레이어입니다. V4 설계 결정 하나가 V5 복잡도를 직접 결정한 핵심 사례입니다.

코드 품질 · 검증

Codex가 코드 품질 1위. V5에서 URLStreamHandlerFactory로 네트워크 레이어를 직접 가로채는 독창적인 테스트 방식을 썼습니다. V6 버그 디버깅에서는 세 AI 중 유일하게 수정 후 단위 테스트까지 직접 실행해 검증했습니다.

V1 한국어 깨짐을 V2에서 스스로 원인을 찾아 수정했습니다. V3에서는 기능 추가 대신 리팩터링을 선택해 코드를 178줄 줄인 유일한 AI입니다. V5 테스트 실패 시 원인을 스스로 진단하고 수정했으며, V7에서는 DIP(의존성 역전 원칙)를 적용해 Repository 인터페이스를 도입했습니다. 전 단계 토큰 효율 1위.

코드 품질 1위 토큰 효율 최고 자가수정 능력
V1–V9 아크: 실패에서 배우는 능력(V2 한국어 수정, V5 테스트 자가진단)과 설계 지향적 선택(V3 리팩터링, V7 DIP)이 가장 두드러집니다. V9에서 FakeOpenMeteoServer 접근법이 세 AI 모두로부터 극찬을 받아 이 실험의 기술적 하이라이트가 됐습니다.

배운 것 1

AI에게 초기 설계를 맡길 때는 테스트 가능성까지 고려해야 합니다. Claude의 V4 Repository 분리 하나로 V5 테스트가 140줄로 끝났지만, Codex의 ViewModel 통합 구조는 245줄이 필요했습니다. AI가 내리는 아키텍처 결정의 파급효과는 즉시 드러나지 않고 이후 단계에서 누적됩니다.

배운 것 2

AI가 만든 코드는 빌드 성공 후 반드시 실기기에서 확인해야 합니다. Gemini V8은 assembleDebug(안드로이드 앱 빌드 명령어) 빌드가 성공했지만 실기기에서 즉시 크래시가 발생했습니다. runBlocking 안티패턴(백그라운드 작업 스레드를 강제로 차단하는 잘못된 코드 패턴)과 플러그인 누락은 컴파일러가 잡지 못하는 유형의 오류입니다.

배운 것 3

각 AI의 강점과 약점은 첫 버전에서 이미 드러납니다. Gemini는 V1 최단 → V8 최단으로 속도 패턴이 일관됐고, Codex는 V1 자동 빌드 → V8 이중 권한 방어로 품질 지향성이 유지됐으며, Claude는 V1 레이어 분리 → V8 정석 패턴으로 설계 일관성이 이어졌습니다. AI를 처음 써볼 때 단순한 과제 하나로도 특성 파악이 가능합니다.

배운 것 4

AI 능력을 제대로 평가하려면 단계별 누적 과제가 필요합니다. V1 한 번만으로는 Codex 한국어 깨짐이 일회성 실수인지 구조적 약점인지 알 수 없습니다. Codex V5 자가수정, Gemini V8 실기기 크래시 자가진단, V9 순환 학습 고리 같은 패턴은 단일 과제에서는 절대 관찰할 수 없습니다.
