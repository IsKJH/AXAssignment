# 카테고리 섹션 통합 스펙 (SCR-05, SCR-06)

> 요구사항: 17:178(SCR-05), 69:251(SCR-06) 텍스트 / UI: 368:1031, 368:1082, 368:1147, 368:1836, 368:1848 (실제 UI 프레임)

## 1. 요구사항 스펙 (SCR-05 카테고리 선택)

### 화면 목적
- SCR-03 카테고리 선택 버튼 탭 시 진입
- 카테고리 선택·선택 해제(미선택) 가능
- 선택 후 SCR-03으로 복귀

### UI 요소
- TopNavigation: 뒤로가기 "<" + "카테고리" (중앙)
- 카테고리 리스트: 이름 + 선택 체크
- 선택된 카테고리: 외곽선 + 컬러 + 체크아이콘
- 카테고리 추가 버튼: 리스트 최하단 고정 → 탭 시 바텀시트(SCR-06) 표시
- 기본 카테고리: 식비, 교통, 쇼핑, 의료, 여가, 주거, 교육 (7개)
- 사용자 추가 카테고리: 수정/삭제 아이콘 표시 (기본 카테고리는 미표시)

### 정책
- 카테고리 탭 → 선택 상태 전환. 이미 선택된 카테고리 재탭 → 선택 해제
- 기본 제공 카테고리(7개)는 수정·삭제 불가
- 카테고리 추가 버튼 탭 → SCR-06 바텀시트 표시 (SCR-05 위에 오버레이)
- 사용자 추가 카테고리 최대 7개 제한. 7개 초과 시 추가 버튼 비활성화
- 카테고리 수정: 수정 아이콘 탭 → SCR-06 바텀시트에 기존 이름 채워진 상태로 표시
- 카테고리 삭제: 삭제 아이콘 탭 → 확인 다이얼로그 표시. 삭제 확인 시 해당 카테고리로 등록된 기존 내역은 "미분류"로 자동 변경

### 플로우
- 카테고리 항목 탭 → 선택 완료 → 이전 페이지 복귀 (값 반영)
- 카테고리 추가 탭 → SCR-06 바텀시트 표시
- 뒤로가기 탭 → 이전 페이지 복귀. 선택 해제 후 뒤로가기 시 미선택 상태로 복귀

## 2. 요구사항 스펙 (SCR-06 카테고리 추가/수정 바텀시트)

### 화면 목적
- SCR-05에서 카테고리 추가 탭 시 올라오는 바텀시트
- 이름 입력 후 저장하면 SCR-05 목록에 추가됨
- 기존 카테고리 수정 시에도 동일 UI 사용 (타이틀과 입력 필드만 변경)

### UI 요소
- 딤 오버레이 (SCR-05 배경)
- 바텀시트: 핸들(드래그 바) + 타이틀 "카테고리 추가하기" 또는 "카테고리 수정하기"
- 이름 입력 필드 (포커스 시 키보드 자동 활성화)
- 저장 버튼 (하단 고정): 이름 입력 시 활성화

### 정책
- 이름 미입력 시 저장 버튼 비활성화
- **중복 이름 입력 시 "이미 있는 카테고리 이름입니다" 인라인 오류 메시지**
- **띄어쓰기 포함, 한/영 최대 12자**
- 저장된 카테고리는 SCR-05 목록 하단에 즉시 추가
- **바텀시트 외부 탭 또는 스와이프 다운 → "카테고리가 저장되지 않았어요" 다이얼로그 → 닫힘 (추가 취소)**
- 바텀시트 진입 시 키보드 자동 활성화

### 플로우
- 저장 탭 → 바텀시트 닫힘 + SCR-05 목록 갱신
- 외부 탭·스와이프 다운 → 다이얼로그 → 바텀시트 닫힘 (SCR-05 유지)

## 3. 실제 UI 스펙 (368:1031 카테고리 설정 메인)

레이아웃: StatusBar → TopBar(56dp, chevron_left 28 + "카테고리" Bold 20sp 중앙) → 리스트
- Section "기본" 라벨 → 기본 7개 항목
- Section "내 카테고리" 라벨 → 사용자 항목(+edit/delete 아이콘) + 카테고리추가 버튼

색상:
- 선택 항목: 배경 #EEF5FF + 외곽선 #559AFF(1.5px) + 텍스트 #559AFF + 체크 아이콘
- 미선택 항목: 배경 white + 외곽선 #EEEEEE(1px) + 텍스트 #272727
- 추가 버튼: 배경 #EEF5FF, 내부 + 아이콘 #559AFF

타이포: 타이틀 Bold 20sp / 섹션 라벨 Medium 14sp / 항목 Regular 16sp(미선택)·Bold 16sp(선택)
치수: 항목 높이 52dp, 너비 328dp, 항목 gap 8dp, 섹션 gap 32dp

## 4. 실제 UI 스펙 (368:1082 추가 바텀시트 / 368:1848 수정 바텀시트)

- Dim: rgba(31,31,31,0.8) 전체
- 시트: 하단 16dp 마진, white, rounded 16dp
- 드래그 핸들: 40x4dp, #D2D2D2
- 타이틀: "카테고리 추가하기" / "카테고리 수정하기" (Bold 18sp)
- 입력 필드: 높이 48dp, 배경 #F7F7FA, 포커스 외곽선 #559AFF 1.5~2px, 플레이스홀더 "새 카테고리 이름을 입력해 주세요" (Regular 14sp)
- 368:1147/1848: 기존 이름("생필품") 채워짐 + X 클리어 버튼
- 저장 버튼: 높이 48dp, 배경 #272727, "저장" Bold 16sp white
- 요소 간 간격 16dp

## 5. 실제 UI 스펙 (368:1836 삭제 확인 다이얼로그)

- 중앙 다이얼로그, white, rounded 16dp
- 제목: "카테고리를 삭제할까요?" (Bold 22sp)
- 설명: "삭제된 내역은 복구할 수 없으니\n신중하게 결정해 주세요." (Medium 14sp, #AFAFAF)
- 버튼 2개 (50%씩, gap 8dp, 높이 48dp): "취소" 배경 #EEEEEE / "삭제" 배경 #DA2E0B white 텍스트

## 6. 핵심 검증 포인트
1. 기본 7개 수정/삭제 아이콘 미표시
2. 선택 시각화: 외곽선+텍스트색+체크
3. 중복 이름 인라인 오류
4. 12자 제한
5. 외부 탭/스와이프 다운 → "카테고리가 저장되지 않았어요" 다이얼로그
6. 사용자 카테고리 최대 7개 → 추가 버튼 비활성화
7. 삭제 시 기존 내역 미분류 전환
