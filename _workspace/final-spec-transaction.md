# 거래내역 섹션 스펙 (SCR-03, SCR-04, 502:1277)

## 요구사항 스펙 전문

### SCR-03: 거래 내역 추가 (16:172)

#### 📌 화면 목적
- 새로운 거래 내역(수입/지출)을 등록하는 화면
- 정기 지출/수입 옵션으로 매달 자동 등록 가능

#### 🔘 UI 요소
- **TopNavigation**: 뒤로가기 + "거래 내역 추가" 제목
- **수입/지출 토글**: 상단에 두 개의 탭 (지출 기본)
- **정기 지출/수입 체크박스**: 탭에 따라 라벨 자동 전환 + ? 아이콘 (툴팁 토글)
  - 툴팁 텍스트: "매달 n일에 자동 등록할 수 있어요." (선택한 일자 기반)
- **금액 입력 필드**: 밑줄 스타일, 플레이스홀더 "금액을 입력해 주세요."
- **지출처/수입처 입력 필드**: 밑줄 스타일, 플레이스홀더 탭에 따라 전환
- **카테고리 선택 행**: 현재값(파란색 #338CFF) + 셰브론
- **지출일시/수입일시 행**: 날짜시간 + 셰브론 (탭에 따라 라벨 전환)
- **확인 버튼**: 하단 고정, 검은색 배경

#### 📐 정책

**활성화 조건**
- 금액 > 0 AND 지출처/수입처 입력 필드 채워짐 → 확인 버튼 활성화
- 금액 0원 OR 지출처/수입처 미입력 → 비활성화

**정기 지출/수입 정책**
- 체크박스 선택 후 저장 시, 등록 월 포함 총 12개월치 미리 생성
- 매월 갱신 시 다음 달 자동 추가 (예: 5월 등록 → 다음 해 4월까지 생성, 6월이 되면 다음 해 5월도 자동 추가)
- 수정 시 범위 선택 다이얼로그: "이 달만 수정" / "이후 내역 모두 수정" / "전체 수정"
- 체크박스 해제 후 저장 → 이후 자동 등록 해제

**날짜·탭 정책**
- 날짜 입력 제한 없음 — 입력 날짜가 속한 주기에 자동 분류
- 지출일시/수입일시 행 탭 → 드럼롤 바텀시트 (날짜·오전오후·시·분)
  - 닫기: 취소 버튼 또는 드래그
  - 완료: 선택값 반영
- 지출일시/수입일시 기본값: FAB 탭 시점의 시간
- 수입/지출 탭 전환 시 금액·지출처·카테고리 등 기입력 값 모두 유지

**카테고리 정책**
- 카테고리 미선택 저장 가능 → "미분류" 처리
- 확인 버튼 활성화: 금액·지출처 입력 시만 (카테고리 무관)
- 통계 탭 그래프에서 별도 항목으로 표시

#### ➡️ 플로우
- 카테고리 선택 행 탭 → SCR-05 카테고리 설정
- 확인 탭 → SCR-02 홈 (저장 완료)
- 뒤로가기 → SCR-02 홈

### SCR-04: 거래 내역 상세 (7:149)

#### 📌 화면 목적
- 선택한 거래 내역의 상세 정보 확인
- 수정 버튼을 통해 편집 모드로 진입
- 정기 내역인 경우 수정/삭제 시 범위 선택 다이얼로그 표시

#### 🔘 UI 요소
- **TopNavigation**: 뒤로가기 + "상세 내역" 제목 + 수정 버튼 (연필 아이콘)
- **정기 표기**: "정기 지출" / "정기 수입" 라벨 (해당 시만 표시)
- **기존 내역 요약 카드**: 금액, 지출처/수입처, 카테고리, 지출/수입일시
- **편집 모드 전환 UI**: 
  - 수정 버튼 탭 시 필드 편집 가능하게 전환
  - 금액, 지출처/수입처 필드에 X 아이콘 추가 (클리어)
  - 카테고리, 지출일시 행은 선택 가능하게 유지
- **저장 버튼**: 편집 모드에서만 표시 (하단 고정)
- **내역 삭제 버튼**: 우측 상단 "내역 삭제" (위험 색상 #D9402E)

#### 📐 정책

**진입 & 읽기 전용**
- TX-002: 리스트 항목 탭 → 상세내역 화면 진입
- 읽기 전용 상태로 먼저 표시, 수정 버튼 탭 시 편집 모드 전환
- 정기 내역인 경우 정기등록 체크박스 체크 상태로 진입

**편집 모드**
- 기존 값이 모두 채워진 상태로 시작
- 금액·지출처 필드는 X 아이콘으로 클리어 가능
- 탭 시 각 필드 편집 가능 상태로 전환

**정기 내역 수정 범위 선택 (바텀시트)**
- 제목: "정기 등록된 내역을 수정할까요?"
- 버튼: "이 달만 수정" / "이후 내역 모두 수정" / "전체 수정" / 취소
- 정책:
  - "이 달만 수정": 현재 주기 내역만 변경, 이후 자동 생성 내역은 원래 값 유지
  - "이후 내역 모두 수정": 현재 주기 포함 이후 자동 생성될 내역에 변경값 적용, 과거 내역 유지
  - "전체 수정": 과거에 생성된 내역 포함 모든 정기 내역에 변경값 적용
- 결과: 토스트 "수정되었습니다." → SCR-02 홈 (잔액 즉시 갱신)

**일반 내역 삭제**
- 내역 삭제 버튼 탭 → 삭제 확인 다이얼로그
  - 제목: "이 내역을 삭제할까요?"
  - 본문: "삭제된 내역은 복구할 수 없으니 신중하게 결정해 주세요."
  - 버튼: "취소" / "삭제" (Danger 색상)
- 결과: SCR-02 홈 (잔액 즉시 갱신)

**정기 내역 삭제 범위 선택 (바텀시트)**
- 제목: "정기 등록된 내역을 삭제할까요?"
- 버튼: "이 달만 삭제" / "이후 내역 모두 삭제" / "전체 삭제" / 취소
- 정책:
  - "이 달만 삭제": 현재 주기 내역만 삭제, 이후 자동 생성 계속 유지
  - "이후 내역 모두 삭제": 현재 주기 포함 이후 자동 생성 중단, 과거 내역 유지 + 정기 지출/수입 해제 처리
  - "전체 삭제": 과거에 생성된 내역 포함 모든 정기 내역 삭제 + 정기 지출/수입 해제 처리
- 결과: SCR-02 홈 (잔액 즉시 갱신)

**뒤로가기 정책**
- 읽기 전용 상태: 바로 이전 화면으로 이동
- 편집 모드 + 편집 내역 있음: "저장하지 않고 나가시겠어요?" 확인 다이얼로그
- 편집 모드 + 편집 내역 없음: 바로 이전 화면으로 이동

#### ➡️ 플로우
- 저장 완료 → SCR-02 홈 (잔액 즉시 갱신)
- 내역 삭제 버튼 탭 → 삭제 확인 다이얼로그
- 편집 모드에서 뒤로가기 → 확인 다이얼로그 (또는 직접 이동)

### 502:1277: 수정 - 정기 등록 해제 (최신 추가)

#### 📌 화면 정의
- **목적**: SCR-04 편집 모드에서 "정기 지출로 등록" 체크박스 해제 후 저장 시 표시되는 확인 다이얼로그
- **시나리오**: 기존 정기 등록 내역을 일반 내역으로 전환할 때의 경고/확인 플로우
- **핵심**: 정기 등록 해제 시 "이후 등록된 내역이 모두 삭제된다"는 경고 메시지

#### 🔘 UI 구성 (스크린샷 기반)

**메인 상세 내역 화면**
- TopBar: 뒤로가기 + "상세내역" + "삭제" (우측)
- 지출/수입 토글 버튼
- **정기 지출로 등록** 체크박스 + ? 아이콘
- 편집 가능한 필드들:
  - 금액: 6,300 + X (클리어 아이콘)
  - 지출처: 이디야 + X (클리어 아이콘)
  - 카테고리 선택 행: 식비 + 화살표
  - 지출일시 행: 5월 29일 16:00 + 화살표
- **저장 버튼** (하단 고정)

**확인 다이얼로그 (최상단 오버레이)**
- Dim 배경 (검은색, 투명도 80%)
- 다이얼로그 박스:
  - **제목**: "정기 등록을 해제할까요?"
  - **본문**: "이후 등록된 내역이 모두 삭제됩니다."
  - **버튼**: 
    - "취소" (회색, #EEE)
    - "등록 해제" (위험 색상, #DA2E0B, 흰색 텍스트)

#### 📐 주요 스펙 & 상태

**체크박스 상태 전환**
- 초기 진입: 체크박스 체크됨 (정기 내역이므로)
- 사용자가 체크박스 해제 → 저장 버튼 탭 → 다이얼로그 표시
- "취소" 버튼: 다이얼로그 닫기, 체크박스 상태 유지 (해제 상태 유지)
- "등록 해제" 버튼: 이후 자동 등록 모두 삭제 + 정기 지출 해제 처리 → SCR-02 홈

**색상 & 스타일**
- TopBar: 흰색 배경, 텍스트 검은색 (#272727)
- 토글: 지출 선택 시 빨강 (#EE3D3D), 수입 비선택 시 회색
- 체크박스: 파란색 테두리 또는 체크 상태 시 파란색 배경
- 다이얼로그 버튼: 취소(#EEE) vs 등록 해제(#DA2E0B, Danger)
- 확인 다이얼로그 배경: 반투명 검은색 (rgba 31, 31, 31, 0.8)

**인터랙션**
- 다이얼로그는 전체 화면 오버레이 (Dim 포함)
- Dim 또는 "취소" 버튼 탭 → 다이얼로그 닫기
- "등록 해제" 버튼 탭 → 즉시 처리 후 홈 화면으로 이동

#### ⚠️ 구현 주의사항

**기존 구현에 없을 가능성 높은 요소**
1. **정기 등록 해제 다이얼로그**: 체크박스 해제 시 확인 다이얼로그 표시
   - 현재 SCR-04 편집 화면에는 "정기 지출로 등록" 체크박스가 있지만, 해제 시 다이얼로그 로직이 빠져있을 수 있음
2. **체크박스 X 아이콘**: 편집 모드에서 금액/지출처 필드에 X 클리어 아이콘 추가 필요
3. **편집 모드 UI**: SCR-04 상세 화면이 읽기 전용 → 편집 모드 전환 로직
   - 수정 버튼 탭 시 필드 활성화, 저장 버튼 표시
4. **다이얼로그 Dim 처리**: 502:1277이 오버레이로 표시되므로, 배경 Dim 처리 필수
5. **토스트 메시지**: 저장 시 "수정되었습니다." 토스트 (SCR-04 정책에 명시)
